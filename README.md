## 系统概述

Polymarket 智能监控系统是一个全自动化的预测市场分析工具，能够：

- 🔍 扫描套利机会（Pair Cost、跨平台套利）
- 🐋 追踪大户（鲸鱼）交易活动
- 📊 分析市场趋势和集中度
- 🔔 自动发送 Telegram 通知
- 📈 生成定期报告

**运行频率**：每5分钟扫描一次

---

## 核心功能

### 1. 套利机会检测

#### Pair Cost 套利
- **原理**：同一事件的 YES + NO 价格之和 < $1.00
- **利润来源**：价格偏差
- **阈值**：Pair Cost < $0.995

#### 跨平台套利
- **平台**：Polymarket ↔ Manifold ↔ Metaculus
- **检测**：同一事件在不同平台的价格差异
- **阈值**：价差 > 5%

### 2. 鲸鱼追踪系统

#### 鲸鱼定义
满足以下任一条件：
- 24h交易量 > $5,000
- 单笔交易 > $500

#### 重点鲸鱼（自动标记）
满足以下任一条件：
- 持仓价值 > $100,000
- 24h变动 > 5次 AND 持仓 > $50,000

### 3. 做市机会扫描

- **数据源**：真实 CLOB 订单簿
- **检测**：买卖价差 > 1.5%
- **过滤**：排除极端价格（<5% 或 >95%）

### 单次扫描完整流程

```
开始扫描 (每5分钟)
    ↓
[1] 获取市场数据
    ├─ 获取Polymarket 500个活跃市场
    ├─ 获取Manifold 100个市场
    └─ 获取Metaculus 50个问题
    ↓
[2] Pair Cost扫描
    ├─ 计算每个市场的YES+NO价格
    ├─ 筛选Pair Cost < $0.995的市场
    └─ Kelly准则评估仓位
    ↓
[3] 跨平台套利扫描
    ├─ 匹配不同平台的相同事件
    ├─ 计算价格差异
    └─ 筛选价差 > 5%的机会
    ↓
[4] 鲸鱼追踪
    ├─ 获取最近1000笔交易
    ├─ 识别活跃鲸鱼(24h交易量>$5k)
    ├─ 分析每个鲸鱼的持仓
    ├─ 检测持仓变化
    ├─ 计算集中度指标(HHI/Top5)
    └─ 更新重点鲸鱼列表
    ↓
[5] 风险评估
    ├─ 对每个机会进行风险评分
    ├─ 过滤异常数据
    └─ 生成建议
    ↓
[6] 发送通知
    ├─ 即时通知(套利/鲸鱼活动)
    └─ 保存报告到JSON
    ↓
结束扫描
```

## 鲸鱼跟踪系统

### 标记说明

| 标记 | 含义 | 触发条件 |
|-----|------|---------|
| 🔥[新重点关注] | 首次加入重点列表 | 满足重点鲸鱼条件 |
| 🔔[重点关注] | 已在重点列表中 | 之前已标记 |
| 🔴 极强 | 信号强度 | 变动 ≥5次 |
| 🟠 强 | 信号强度 | 变动 3-4次 |
| 🟡 中等 | 信号强度 | 变动 1-2次 |

### 收敛检测

系统自动检测鲸鱼策略是否从分散→集中：

| 状态 | Top5占比 | 说明 |
|-----|---------|------|
| 高度分散 | <30% | 持仓分散在多个市场 |
| 适度分散 | 30%-50% | 开始形成重点 |
| 开始集中 🔥 | 50%-70% | 策略明确化 |
| 高度集中 | >70% | 强信号 |

**趋势检测**：
- 🔥**正在收敛！** - Top5占比增加 >10%
- 正在分散 - Top5占比减少 >10%
- 稳定 - 变化 <10%

## 故障排除

### 常见问题

#### 1. 没有收到通知
- 检查 `TELEGRAM_BOT_TOKEN` 和 `TELEGRAM_CHAT_ID`
- 查看日志：`tail -f 07-data/logs/monitor.log`

#### 2. Metaculus API 错误
- 正常现象，SSL间歇性问题
- 不影响核心功能（Polymarket + Manifold）

#### 3. 异常数据（持仓多但价值低）
- 已自动过滤（持仓>50且价值<$1000）
- 控制台显示 "⚠️ 数据异常，跳过"

#### 4. 测试数据污染
- 已修复：测试必须使用 `--test` 参数
- 测试数据使用 `/tmp/` 临时文件

### 日志查看
