## 🎯 设计原则

| 原则 | 说明 |
|-----|------|
| **针对性** | 只抓取与重点鲸鱼持仓相关的新闻 |
| **轻量级** | 不存储历史，只保留最近24小时 |
| **触发式** | 只在发现重点鲸鱼时启动 |
| **辅助性** | 作为参考信息，不自动决策 |

---

## 📋 功能设计

### 1. 触发条件

```
当发现重点鲸鱼（如Stupendous-Eddy）时：
├─ 提取其持仓关键词（Iran、Israel、Hormuz...）
├─ 启动新闻抓取
└─ 抓取最近24小时相关新闻
```

### 2. 抓取来源

| 来源 | 内容 | 优先级 |
|-----|------|--------|
| **Twitter/X** | 相关话题、KOL发言 | 高 |
| **新闻API** | 主流新闻网站 | 中 |
| **Reddit** | r/politics、r/worldnews | 低 |

### 3. 输出格式示例

```
🐋 Stupendous-Eddy 持仓新闻

📊 持仓主题: 伊朗局势
🔍 关键词: Iran, Israel, Strait of Hormuz

📰 相关新闻 (最近6小时):
1. [2小时前] Reuters: 伊朗表示愿意恢复核谈判
   🔗 链接
   
2. [4小时前] Bloomberg: 美国加强海湾地区军事部署
   🔗 链接

💡 情绪分析:
- 正面: 1条 (谈判意愿)
- 负面: 1条 (军事部署)
- 中性: 0条

⚠️ 提醒: 新闻仅供参考，不构成投资建议
```

## 🔧 技术方案对比

### 方案A：简单版（推荐先做这个）

**功能**:
- 手动触发：输入关键词，抓取新闻
- 输出：新闻标题+链接+时间
- 集成：在鲸鱼警报后自动触发

**技术**:
- RSS订阅主流新闻源
- Twitter API抓取推文
- 简单关键词匹配

**优点**:
- 快速实现（1-2小时）
- 不增加系统复杂度
- 验证是否有用
- 有用再升级

### 方案B：增强版（未来考虑）

**功能**:
- 自动情绪分析
- 关联度评分
- 历史对比
- 生成策略建议

**技术**:
- LLM分析新闻内容
- 向量数据库
- 机器学习模型
