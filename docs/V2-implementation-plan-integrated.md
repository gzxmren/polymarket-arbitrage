## 三、依赖关系图

```
Phase 1: 语义理解（核心壁垒）
    ├─→ Phase 2: 行为预测（差异化）
    │       ├─→ Phase 3: 质量监控（数据驱动）
    │       │       └─→ Phase 4: 历史优化（持续改进）
    │       │
    └───────┴─→ Phase 5: 跨平台（扩展）
```

**关键依赖**:
- 必须先有信号，才能追踪效果（Phase 2 → Phase 3）
- 必须先有效果数据，才能历史分析（Phase 3 → Phase 4）
- 语义理解是独立能力，可并行

## 九、总结

### 核心变化
1. **新增Phase 1**: 语义套利 + 逻辑链矛盾检测（最高优先级）
2. **调整Phase 2-4**: 延后1周，为语义能力让路
3. **明确依赖**: 先建立壁垒，再扩展功能

### 关键决策
- ✅ **做**: 语义套利、逻辑链矛盾、鲸鱼预测、质量监控
- ⚠️ **延后**: 跨平台套利、Dashboard细节优化
- ❌ **不做**: 毫秒级数学套利（红海）

### 预期差异化
- **现有机器人**: 数字驱动，无法理解语义
- **我们的系统**: LLM驱动，理解市场关系，识别逻辑矛盾
