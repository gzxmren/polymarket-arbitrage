## 二、术语定义

| 术语 | 定义 |
|------|------|
| 语义套利 | 利用语义相关市场之间的定价矛盾进行套利 |
| 逻辑链矛盾 | 逻辑上存在蕴含关系的市场之间的定价违反数学规律 |
| 聪明钱鲸鱼 | 历史胜率 >60%，夏普比率 >0.5 的高价值鲸鱼 |
| 信号 | 系统生成的交易建议，包含方向、仓位、置信度 |
| 背离 | 新闻情绪与市场价格方向相反的情况 |

## 三、功能需求清单

### 模块 1: 语义套利系统 (Semantic Arbitrage) 🔴 P0

#### 1.1 功能描述
使用 LLM 识别语义相关的预测市场，检测定价矛盾，生成套利信号。

#### 1.2 用户故事
- 作为交易者，我希望系统能自动发现"AFC赢得超级碗"和"酋长队赢得超级碗"之间的定价矛盾
- 作为交易者，我希望系统能识别语义相关市场之间的套利机会

#### 1.3 功能需求

**FR-1.1.1** 市场语义相似度计算
- 输入: 两个市场的标题和描述
- 处理: 使用 LLM 计算语义相似度
- 输出: 相似度评分 (0-1) 和关系类型
- 验收标准: 相似度 >0.8 的市场对准确率 >90%

**FR-1.1.2** 逻辑关系识别
- 支持的关系类型:
  - `implies`: A 发生则 B 必然发生 (A⊆B)
  - `mutex`: A 和 B 互斥 (A∩B=∅)
  - `related`: 相关但无明确逻辑关系
- 验收标准: 逻辑关系识别准确率 >85%

**FR-1.1.3** 定价矛盾检测
- 检测规则:
  - 如果 A implies B，则 Price(A) ≤ Price(B) + 5%容错
  - 如果 A mutex B，则 Price(A) + Price(B) ≤ 100% + 5%容错
- 输出: 矛盾详情和套利建议
- 验收标准: 矛盾检测准确率 >95%

**FR-1.1.4** 语义套利信号生成
- 信号内容:
  - 矛盾类型 (implies_violation / mutex_violation)
  - 涉及市场
  - 当前定价
  - 套利操作 (SELL A, BUY B)
  - 预期收益
  - 置信度
- 验收标准: 信号格式符合规范

#### 1.4 非功能需求
- **NF-1.1.1** LLM 调用延迟 <3秒
- **NF-1.1.2** 支持每日1000+市场的语义分析
- **NF-1.1.3** LLM API 成本 <$200/月

### 模块 2: 逻辑链矛盾检测 (Logic Chain Analyzer) 🔴 P0

#### 2.1 功能描述
预定义常见逻辑关系，自动检测跨市场的定价矛盾。

#### 2.2 用户故事
- 作为交易者，我希望系统能自动发现"Trump赢得2024"和"共和党赢得2024"之间的定价矛盾
- 作为交易者，我希望系统能维护一个逻辑关系数据库

#### 2.3 功能需求

**FR-2.1.1** 逻辑关系数据库
- 预定义关系:
```python
IMPLICATIONS = {
    "Trump wins 2024": ["Republican wins 2024", "Male wins 2024"],
    "Chiefs win Super Bowl": ["AFC wins Super Bowl"],
    "Iran regime falls": ["Middle East regime change"],
    # ... 持续扩展
}
```
- 验收标准: 覆盖Top 100热门事件

**FR-2.1.2** 自动逻辑关系发现
- 使用 LLM 自动发现新的逻辑关系
- 人工审核后入库
- 验收标准: 每周发现5+新关系

**FR-2.1.3** 实时矛盾扫描
- 扫描频率: 每10分钟
- 检测所有预定义关系
- 生成矛盾警报
- 验收标准: 延迟 <1分钟

**FR-2.1.4** 矛盾信号生成
- 信号格式:
```
🔴 逻辑链矛盾 detected!

关系: Chiefs win Super Bowl → AFC wins Super Bowl
矛盾: Chiefs(50%) > AFC(45%)

套利操作:
- SELL Chiefs win @ 50%
- BUY AFC win @ 45%
- 预期收益: 5%

置信度: 95%
```

#### 2.4 非功能需求
- **NF-2.1.1** 扫描延迟 <30秒
- **NF-2.1.2** 误报率 <5%

### 模块 3: 鲸鱼跟随策略 (Whale Following) 🟢 P1

#### 3.1 功能描述
识别聪明钱鲸鱼，跟踪大额调仓，生成跟随信号。

#### 3.2 用户故事
- 作为交易者，我希望知道哪些鲸鱼是"聪明钱"
- 作为交易者，我希望在鲸鱼大额调仓时收到通知
- 作为交易者，我希望知道应该跟随多少仓位

#### 3.3 功能需求

**FR-3.1.1** 聪明钱识别
- 识别标准:
  - 历史胜率 >= 60%
  - 交易次数 >= 10
  - 夏普比率 >= 0.5
  - 策略一致性 >= 0.6
- 输出: 聪明钱鲸鱼列表
- 验收标准: 准确率 >70%

**FR-3.1.2** 大额调仓检测
- 检测阈值: >$10k
- 检测时间窗口: 1小时内
- 输出: 调仓详情
- 验收标准: 实时检测，延迟 <5分钟

**FR-3.1.3** 跟随信号生成
- 置信度计算:
  - 历史胜率: 40%
  - 行为一致性: 30%
  - 市场规模: 20%
  - 时机: 10%
- 建议仓位: 鲸鱼仓位的 1%-5%
- 风险等级: 低/中/高
- 验收标准: 信号胜率 >60%

**FR-3.1.4** 信号追踪
- 记录信号发出时的价格
- 48小时后检查实际结果
- 更新胜率统计
- 验收标准: 追踪率 100%

### 模块 4: 新闻驱动策略 (News Driven) 🟢 P1

#### 4.1 功能描述
抓取相关新闻，分析情绪，检测与价格的背离，生成信号。

#### 4.2 用户故事
- 作为交易者，我希望知道持仓相关的新闻
- 作为交易者，我希望发现新闻情绪与价格的背离
- 作为交易者，我希望在重大新闻发布时收到通知

#### 4.3 功能需求

**FR-4.1.1** 新闻抓取
- 数据源: BBC RSS (已接入)
- 更新频率: 每30分钟
- 关联度评分: 0-1
- 验收标准: 覆盖率 >80%

**FR-4.1.2** 情绪分析
- 使用 LLM 分析新闻情绪
- 情绪类型: 正面 / 负面 / 中性
- 置信度评分
- 验收标准: 准确率 >75%

**FR-4.1.3** 背离检测
- 背离定义:
  - 正面新闻 + 价格 <40% = 做多机会
  - 负面新闻 + 价格 >60% = 做空机会
- 背离度计算
- 验收标准: 背离检测准确率 >70%

**FR-4.1.4** 新闻信号生成
- 信号内容:
  - 新闻标题和摘要
  - 情绪评分
  - 当前价格
  - 背离度
  - 建议操作
- 验收标准: 信号胜率 >65%

### 模块 5: 鲸鱼行为预测 (Behavior Prediction) 🟢 P1

#### 5.1 功能描述
分析鲸鱼历史行为，预测下一步动作。

#### 5.2 用户故事
- 作为交易者，我希望预测鲸鱼下一步会做什么
- 作为交易者，我希望在鲸鱼大幅调仓前收到预警

#### 5.3 功能需求

**FR-5.1.1** 行为模式识别
- 识别特征:
  - 调仓频率突增
  - 集中度从分散→集中
  - 连续同方向调仓
- 输出: 行为模式标签
- 验收标准: 识别准确率 >60%

**FR-5.1.2** 预测模型
- 预测时间窗口: 48小时
- 预测内容: 调仓概率、方向、规模
- 置信度评分
- 验收标准: 预测准确率 >55%

**FR-5.1.3** 预警生成
- 触发条件: 预测概率 >70%
- 预警内容:
  - 鲸鱼信息
  - 预测动作
  - 置信度
  - 建议准备
- 验收标准: 预警准确率 >50%

### 模块 6: 质量监控系统 (Quality Monitor) 🟢 P1

#### 6.1 功能描述
追踪信号效果，生成质量报告，实现数据驱动的优化。

#### 6.2 用户故事
- 作为交易者，我希望知道信号的实际效果
- 作为交易者，我希望收到每周质量报告
- 作为交易者，我希望系统能自动优化策略

#### 6.3 功能需求

**FR-6.1.1** 信号效果追踪
- 追踪字段:
  - 发出时间
  - 预期方向
  - 预期价格变化
  - 实际结果
  - 实际盈亏
- 追踪周期: 48小时
- 验收标准: 追踪率 100%

**FR-6.1.2** 胜率统计
- 统计维度:
  - 整体胜率
  - 分策略胜率
  - 分市场胜率
  - 分时段胜率
- 更新频率: 实时
- 验收标准: 数据准确性 100%

**FR-6.1.3** 每周质量报告
- 报告内容:
  - 信号总数
  - 胜率统计
  - 收益曲线
  - 策略对比
  - 预警与建议
- 发送方式: Telegram + Dashboard
- 发送时间: 每周一 09:00
- 验收标准: 自动生成，无人工干预

**FR-6.1.4** 质量预警
- 预警规则:
  - 胜率 <50%: 🔴 红灯
  - 3天无信号: 🔴 红灯
  - 误报率 >30%: 🟡 黄灯
- 通知方式: Telegram
- 验收标准: 实时预警

    │
    ├─→ Phase 2 (鲸鱼+新闻+预测)
    │       │
    │       ├─→ Phase 3 (质量监控)
    │       │       │
