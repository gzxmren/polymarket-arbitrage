# 程序质量评估体系

## 📊 核心指标

### 1. 发现能力指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **扫描覆盖率** | 每小时扫描市场数 | > 200个 |
| **机会发现率** | 每周发现套利机会数 | > 0个 |
| **跨平台匹配率** | 成功匹配事件比例 | > 5% |
| **鲸鱼识别率** | 识别活跃大户比例 | > 1% |

### 2. 准确性指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **风险评估准确率** | 通过审核后实际盈利比例 | > 70% |
| **误报率** | 无效警报比例 | < 20% |
| **漏报率** | 遗漏真实机会比例 | < 10% |

### 3. 系统稳定性指标

| 指标 | 说明 | 目标值 |
|------|------|--------|
| **系统可用性** | 正常运行时间 | > 99% |
| **通知成功率** | Telegram消息送达率 | > 95% |
| **扫描成功率** | API调用成功率 | > 98% |

## 📅 每周质量Review

### 周一：数据收集
- 导出上周所有日志
- 统计核心指标
- 记录发现的机会

### 周三：质量分析
- 分析发现的机会质量
- 检查误报/漏报情况
- 评估风险评估准确性

### 周五：改进计划
- 识别程序缺陷
- 制定优化方案
- 更新AGENTS.md

## 📋 Review检查清单

### 发现能力检查
- [ ] 扫描是否覆盖所有活跃市场？
- [ ] 跨平台匹配是否准确？
- [ ] 鲸鱼追踪是否及时？
- [ ] 是否遗漏明显机会？

### 准确性检查
- [ ] 风险评估是否合理？
- [ ] 通过的审核是否盈利？
- [ ] 拒绝的机会是否真的差？
- [ ] 通知内容是否清晰无误？

### 稳定性检查
- [ ] 程序是否持续运行？
- [ ] 通知是否正常发送？
- [ ] 日志是否完整记录？
- [ ] 错误是否及时处理？

## 🚨 预警机制

### 红灯预警（立即处理）
- 连续3天未发现任何机会
- 系统可用性 < 95%
- 风险评估准确率 < 50%

### 黄灯预警（需要关注）
- 连续1周机会数 < 2个
- 误报率 > 30%
- 通知成功率 < 90%

### 绿灯正常
- 各项指标达标
- 程序稳定运行
